Carolina Digital Repository

While I worked at UNC Chapel Hill, I was with The Repository Application Team to manage the digital repositories for serving the university communities.

The Great Migration

Up until 2019, both the academic contents and the libraries' historical archives lived in a monolithic repository system — this was a source of growing pains in terms of content discovery and usability, and the situation only worsened as more and more users added .

Thus came "The Great Migration" initiative.

At the highest level, the inititiave aimed to address the following problems:

  • Tangled use cases that confuse users of academic vs archival contents
  • Limited legacy software system that make adding new features challenging
  • Dense information architecture that leads to disorganization of contents

The answer was to migrate the academic contents to a new open source repository system and have it inherit the name of Carolina Digital Repository (CDR 2.0).

Without having to devise workarounds, the renamed legacy system — Digital Collections Repository (DCR) — can be optimized and more easily maintained in the long run.

VALIDATION A Task-Oriented Usability Test of CDR 2.0

As a part of my own academic research, I conducted a formal usability test with the target users. This was done with the alpha release of the new CDR 2.0 platform, and the findings were then used to introduce specific workflow improvements to help better prepare for the public launch.

DESIGN A New Look to the Legacy Platform

For the DCR legacy platform, almost all of the design efforts went into addressing age-old issues that were overlooked throughout the years of ad-hoc development.

Additionally, some of the effort went into developing a concept for more intuitive and streamlined access controls for the repository administrators.
